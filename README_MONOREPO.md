
# Mark & Weld Monorepo 规范

本仓库采用单仓库多项目架构，旨在共享核心工业逻辑。

## 目录结构
- `/core`: **核心逻辑区**。包含数学模型、DXF 解析、数据类型。任何 App 严禁在此目录编写 UI 代码。
- `/apps/design-app`: **设计端**。负责图片测量、标定、特征点位导出。
- `/apps/weld-app`: **生产调试端**。负责焊机坐标校验、运动路径模拟（开发中）。

## 开发规则
1. **单向依赖**: App 引用 Core，Core 不得引用 App。
2. **核心保护**: 修改 `/core` 代码需考虑兼容性，确保不会破坏另一个 App 的运行。
3. **数据一致性**: 必须统一使用 `/core/types.ts` 定义的 `ProjectConfig` 进行 JSON 解析，确保设计端导出的配置能被生产端完美加载。
